Allocation setup

How this is computed

Each block is a fixed-size partition that can hold at most one process. Processes are placed in the order given. First Fit takes the first block large enough; Best Fit takes the smallest block large enough; Worst Fit takes the largest available block. Leftover space in an occupied block is internal fragmentation.

Enter block and process sizes, then run the allocation to see the memory map and fragmentation.

Request setup

How this is computed

SCAN travels to the disk boundary in its current direction (even if no request sits exactly there), then reverses and services the rest, stopping once every request is served. C-SCAN also jumps back to cylinder 0 after reaching the far boundary, and that return jump is counted as real head movement. These are the standard, physically-accurate definitions — some course slides simplify by skipping the boundary trip or treating the jump as free, so your instructor's numbers may differ slightly depending on which convention they teach.
